Web Research & Debugging

1. Fixing Canvas Rendering Lag in Tetris

Problem: Canvas flickered or lagged during gameplay.

What I Researched: Optimizing canvas rendering in JavaScript games.

Resource: MDN Web Docs - Canvas Optimization

Improved Code Snippet:

function updateCanvas() {
    context.clearRect(0, 0, canvas.width, canvas.height);  
    drawArena();
    drawPlayer();
}

Why It Matters: Reduced unnecessary re-renders and improved performance.

2. Disabling Letter Buttons After Game Ends (Hangman)

Problem: Letter buttons could be clicked even after game ended.

What I Researched: How to disable buttons dynamically with JavaScript.

Resource: StackOverflow - Disable Buttons

Code Snippet:

function disableAllButtons() {
    const buttons = document.querySelectorAll(".letter-button");
    buttons.forEach(button => button.disabled = true);
}

Why It Matters: Prevents invalid input after game conclusion.

3. Styling Error Messages for Form Validation

Problem: Plain error text with no style cluttered the form.

What I Researched: Styling and organizing form validation messages.

Resource: CSS-Tricks - Form Validation UX

JavaScript Code:

function showError(input, message) {
    const errorList = input.nextElementSibling;
    errorList.innerHTML = `
  • ${message}
    • `;
    errorList.classList.add("error-visible");
}

    CSS Snippet:

    .error-visible {
    color: red;
    font-size: 0.9em;
    margin-top: 5px;
}

    Why It Matters: Cleaner, readable feedback improves UX and accessibility.
