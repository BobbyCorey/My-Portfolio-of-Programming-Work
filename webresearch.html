Debugging & Web Research

As part of my development journey, I frequently encountered bugs and roadblocks that required in-depth debugging and research. Here are a couple of real examples that highlight how I tackled those situations:

1. Git LFS Issues with Unity Project

Problem: When pushing my Unity Paintball Game to GitHub, I ran into errors related to large file sizes.

Debug Process: I researched Git Large File Storage (LFS), installed it, and then tracked `.unitypackage` and other large asset files using `git lfs track "*.ext"` syntax.

Solution: I cleaned my local repository and pushed again with LFS enabled, which resolved the push failure. I also updated `.gitattributes` to prevent future size issues.

2. VR Voice Recognition Failure (Quest 1)

Problem: My Meta Quest 1 project failed to recognize speech using Android’s native SpeechRecognizer API — it lacked the required voice service.

Debug Process: After extensive forum and API documentation research, I discovered Quest 1 doesn’t ship with Google Voice Services by default. I explored alternatives like using Wit.ai via REST API for runtime recognition without UI prompts.

Solution: I switched to a continuous listening setup using Wit.ai’s API to stream microphone input and parse results manually. This removed reliance on missing services and restored voice interaction features.

Takeaway

Debugging and researching are essential skills in software development. These experiences not only improved my technical proficiency but also taught me how to break down complex problems and find reliable solutions independently.
