Hangman Game

Can you guess the word before the hangman is fully drawn?

This Hangman game was built using HTML, CSS, and JavaScript. The game uses JavaScript classes for managing the game state and handles player interaction through a responsive GUI. DOM manipulation updates the board dynamically as the user guesses letters.

This project helped reinforce concepts such as conditionals, loops, event listeners, and modular design. No database was used; all word data is stored in-memory.

Inspiration came from classic word-guessing games. I researched how to create modal popups for images, and how to manage game state using JS classes (see code below).

Here’s a class-based structure used to manage the game state:


// GUI logic and game state
class HangmanGame {
  constructor(wordList) {
    this.wordList = wordList;
    this.resetGame();
  }

  resetGame() {
    this.word = this.getRandomWord();
    this.correctGuesses = [];
    this.incorrectGuesses = [];
  }

  getRandomWord() {
    const index = Math.floor(Math.random() * this.wordList.length);
    return this.wordList[index];
  }

  checkGuess(letter) {
    if (this.word.includes(letter)) {
      this.correctGuesses.push(letter);
    } else {
      this.incorrectGuesses.push(letter);
    }
  }
}

This approach demonstrates the use of JavaScript classes, inheritance-ready structure, and encapsulation of logic.
